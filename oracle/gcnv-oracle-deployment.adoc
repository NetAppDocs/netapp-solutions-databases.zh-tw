= 使用 Google Cloud NetApp Volumes 工作流程部署高可用性 Oracle Database 26ai
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
使用 Google Cloud NetApp Volumes iSCSI 區塊存儲，在 Google Compute Engine 上部署具有 Oracle Data Guard 的高可用性 Oracle Database 26ai。本工作流程將引導您完成部署選項、基礎架構配置、Oracle 軟體安裝和高可用性配置，並利用 GCNV 的儲存層複製功能加速備用資料庫的初始化。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["一"]link:gcnv-oracle-solution-architecture.html["檢閱解決方案架構"]
[role="quick-margin-para"]
了解 Google Cloud NetApp Volumes 的 Oracle Database 26ai 高可用性架構，包括 GCNV iSCSI 區塊儲存設計、Data Guard 整合和儲存拓撲。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["二"]link:gcnv-oracle-deployment-options.html["檢閱部署選項與需求"]
[role="quick-margin-para"]
比較從開發/測試到生產高可用性的部署層級，並確認基礎架構、存取和軟體先決條件。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["三"]link:gcnv-oracle-access-and-apis.html["驗證存取和 API"]
[role="quick-margin-para"]
確認所需的存取、API 啟用、網路先決條件和時間同步。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["四"]link:gcnv-oracle-provision-compute.html["佈建 Google Compute Engine 執行個體"]
[role="quick-margin-para"]
建立主資料庫、備用資料庫和快速啟動故障轉移觀察器的虛擬機器。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["五"]link:gcnv-oracle-provision-storage.html["佈建 GCNV iSCSI 儲存設備"]
[role="quick-margin-para"]
建立 GCNV 儲存池和 iSCSI 磁碟區、設定多重路徑，以及設定 ASM 磁碟群組。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png["六"]link:gcnv-oracle-install-software.html["安裝 Oracle Grid Infrastructure 和 Oracle Database 26ai"]
[role="quick-margin-para"]
在兩台資料庫主機上安裝 Oracle Grid Infrastructure 和 Oracle Database 26ai 軟體。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-7.png["七"]link:gcnv-oracle-primary-database.html["建立主資料庫"]
[role="quick-margin-para"]
建立具有 ASM 儲存的主要 Oracle 資料庫，並設定 Oracle Restart。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-8.png["八"]link:gcnv-oracle-standby-database.html["建立備用資料庫"]
[role="quick-margin-para"]
使用 GCNV 複寫、快照或複製初始化備用資料庫，然後向 Oracle Restart 註冊。所有 HA 層級都需完成此步驟。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-9.png["九"]link:gcnv-oracle-standby-finalize-dg.html["完成 Data Guard 的備用資料庫配置"]
[role="quick-margin-para"]
建立備用重做日誌，啟用閃回，啟動重做傳輸，並驗證 Data Guard 狀態。*層級特定：* 僅生產高可用性（Data Guard + FSFO）層級需要。

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-10.png["十"]link:gcnv-oracle-data-guard.html["設定 Data Guard Broker 和 Fast-Start Failover"]
[role="quick-margin-para"]
啟用代理，建立 Data Guard 組態，啟用 FSFO，並設定 Observer。*層級特定：* 僅生產高可用性（Data Guard + FSFO）層需要。
